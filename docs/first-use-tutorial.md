## 一、什么是 QuickDock？

**快启坞 QuickDock** 是一款面向 Windows 开发者的桌面效率工具。它的设计灵感来源于：

- **Raycast / Spotlight** — 快捷键唤出，快速搜索和执行
- **VS Code** — 工作空间概念，按项目组织资源

QuickDock 帮你把**项目目录、常用网页、终端命令、应用程序、文本片段、剪贴板历史**全部整合到一个统一的界面中，配合全局热键随时随地唤出，不再需要在多个窗口之间来回切换。

### 谁适合用？

- **开发者**：管理多个项目，保存常用命令和目录
- **设计师 / 产品经理**：集中存放设计稿链接、规范文档
- **效率控**：习惯用键盘操作电脑、离不开剪贴板历史的用户

## 二、快速上手

### 安装

1. 运行安装包 `quickdock-amd64-installer.exe`
2. 按照安装向导完成安装
3. 安装完成后自动启动，QuickDock 图标会出现在系统托盘（任务栏右下角）

### 启动与基本布局

启动后你会看到主界面，分为以下几个区域：

```
┌─────────────┬──────────────────────────────┐
│  左侧导航栏  │         主内容区              │
│             │                              │
│  ◎ 工作空间 │  [场景标签页]                 │
│  ◇ 场景     │  [集合列表]                   │
│             │  [项目列表]                   │
│  ─────────  │                              │
│  📋 剪贴板  │                              │
│  📝 片段    │                              │
│  ✅ 待办    │                              │
│  ⏰ 定时    │                              │
│  📊 监控    │                              │
│  📦 插件    │                              │
│  ⚙️ 设置    │                              │
└─────────────┴──────────────────────────────┘
```

- **左侧导航栏**：切换不同功能模块
- **顶部场景标签**：在当前工作空间内切换不同场景
- **中间内容区**：显示当前模块的具体内容

### 创建你的第一个工作空间

首次启动时，如果还没有工作空间，系统会引导你创建第一个工作空间。也可以手动创建：

1. 点击左侧导航栏顶部的 **"工作空间"** 区域
2. 点击 **"新建工作空间"**
3. 输入名称（如"我的工作"）
4. 点击确定

> 💡 **工作空间是最顶层容器**，可以把不同性质的工作分开管理（比如一个用于工作、一个用于个人项目）。

## 三、核心概念：三步结构

QuickDock 的组织方式是一个 **三层结构**：

```
工作空间（Workspace）          ← 最顶层，一个独立的工作上下文
  └── 场景（Scene）            ← 按项目或功能分组
       └── 集合（Collection）   ← 按类型或用途归类
            └── 项目（Item）    ← 具体的资源（目录/网页/命令等）
```

### 工作空间（Workspace）

每个工作空间是一个**完全独立的环境**。你可以创建多个：

- `工作` — 存放工作相关的所有资源
- `个人项目` — 存放个人项目、学习资料
- `博客维护` — 专门管理博客相关的内容

**切换工作空间**：点击左侧导航栏顶部的工作空间名称，在下拉菜单中切换。

### 场景（Scene）

场景是**工作空间下的视图分组**，类似浏览器的标签页：

- `日常开发` — 开发中最常用的资源和工具
- `项目管理` — 项目文档、甘特图链接、会议记录
- `部署运维` — 服务器地址、部署指南、监控地址

**管理场景**：在主界面上方的标签栏，点击 `+` 新建场景，右键场景标签可以关闭或排序。

### 集合（Collection）

集合是**更细粒度的分类**，把同一类资源放到一起：

- `常用工具` — 编辑器、终端、设计软件
- `项目文档` — PRD、设计稿、API 文档链接
- `快捷命令` — git 提交流程、构建脚本、数据库连接

**创建集合**：选中场景后，在中间列表点击 `+`，填写名称并选择集合类型。

**集合类型**：目录集合、网页集合、命令集合、文件集合、应用集合

### 项目（Item）

项目是**最具体的资源**，支持 6 种类型：

| 类型 | 说明 | 示例值 |
|------|------|--------|
| `directory` 目录 | 打开文件夹 | `D:\project\quickdock` |
| `file` 文件 | 打开文件 | `D:\docs\readme.md` |
| `url` 网页 | 在浏览器中打开 | `https://github.com` |
| `command` 命令 | 在终端中执行 | `git status` |
| `app` 应用 | 启动应用程序 | `C:\Windows\System32\notepad.exe` |
| `quicklink` 快速链接 | 带参数的快捷方式 | 支持 `{query}` 占位符 |

> 🔗 **快速链接的高级用法**：在值中使用 `{query}` 占位符，在命令面板中选中该链接时会提示输入内容，自动替换 `{query}`。例如一个 Google 搜索链接：`https://www.google.com/search?q={query}`

### 实战：搭建一个开发工作空间

用一个实际例子走一遍流程：

#### 1. 创建"后端开发"工作空间

- 点击导航栏顶部 → 新建工作空间 → 起名：`后端开发`

#### 2. 添加场景

- 添加场景 `日常开发`
- 添加场景 `项目文档`

#### 3. 在"日常开发"场景下添加集合

- `常用工具`（类型：应用集合）
- `项目目录`（类型：目录集合）
- `常用命令`（类型：命令集合）

#### 4. 在集合中添加项目

- 在"常用工具"下添加：
  - VS Code（类型：app，路径：`C:\Users\你\AppData\Local\Programs\Microsoft VS Code\Code.exe`）
  - Chrome（类型：app）
- 在"项目目录"下添加：
  - 后端项目（类型：directory，路径：`D:\project\quickdock`）
- 在"常用命令"下添加：
  - Git 状态（类型：command，值：`git status`）
  - 构建项目（类型：command，值：`go build ./...`）

> 💡 以后每次打开 QuickDock，直接双击项目就能一键跳转目录、打开应用或执行命令。

## 四、全局热键一览

QuickDock 的精髓在于**键盘驱动**。默认全局热键：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl + Space` | 切换主窗口 | 显示/隐藏主界面 |
| `` Ctrl + ` `` | 切换剪贴板窗口 | 显示/隐藏剪贴板浮动窗口 |
| `Ctrl + K` | 切换命令面板 | 显示/隐藏命令面板 |
| `Ctrl + Shift + N` | 切换快捷笔记 | 显示/隐藏笔记浮动窗口 |

> 所有热键都可以在 **设置 > 快捷键** 页面自定义，避免与其他软件冲突。

## 五、各功能模块详解

### 1. 命令面板（Command Palette）

命令面板是 QuickDock 的**中枢神经**，按 `Ctrl+K` 随时唤出。

#### 能搜什么？

- 工作空间、场景、集合、项目的名称
- 文本片段（Snippets）
- 系统命令（锁屏、关机等）
- 内置插件命令
- 最近使用的记录

#### 高级功能

- **数学计算 / 单位换算**：直接输入 `1024MB = ?GB` 或 `100kg = ?lb`
- **URL 智能建议**：复制链接后打开命令面板，自动提示打开或保存
- **多选批量操作**：按住 `Ctrl` 点击多选，按 `Enter` 批量打开
- **{query} 占位符**：选中含占位符的项目后输入内容

#### 使用技巧

```
# 查找项目
输入 "展开" → 搜索所有包含关键词的资源

# 快速计算
输入 "1024MB = ?GB" → 显示换算结果

# 批量打开网页
按住 Ctrl 点击多个网页链接 → Enter 全部打开
```

---

### 2. 剪贴板历史（Clipboard History）

按 `` Ctrl+` `` 唤出剪贴板浮动窗口，自动记录你复制过的内容。

#### 功能

- **文本**：自动记录所有复制的文字
- **图片**：复制图片后自动保存到历史
- **文件**：记录复制过的文件路径
- **搜索**：在剪贴板历史中搜索
- **固定**：常用内容点击星标固定，不被清除
- **一键复制**：点击任一记录即复制回剪贴板

#### 数据保留

剪贴板历史保留最近 **30 天**（可在设置中调整），超过后自动清理。

---

### 3. 快捷笔记（Quick Notes）

按 `Ctrl+Shift+N` 唤出独立笔记窗口，随时记录灵感。

- **自动保存**：停止输入 500ms 后自动保存，无需手动操作
- **内容安全**：笔记数据存储在本地，不经过任何云端
- **随时唤出**：独立浮动窗口，不打断当前工作

---

### 4. 文本片段（Snippets）

预先保存常用文本，一键粘贴。

#### 适用场景

- 常用的代码片段（如 git 提交流程、SQL 查询模板）
- 邮件模板（周报、请假、感谢信）
- 项目名称、版本号、标准回复

#### 变量替换

片段内容支持动态变量：

| 变量 | 说明 |
|------|------|
| `{date}` | 当前日期，格式：2026-07-17 |
| `{time}` | 当前时间，格式：14:30 |
| `{username}` | Windows 用户名 |
| `{clipboard}` | 当前剪贴板内容 |

**示例**：在片段中写 `git commit -m "fix: {date} 修复问题"`，粘贴时自动替换为 `git commit -m "fix: 2026-07-17 修复问题"`。

---

### 5. 待办任务（Todos）

轻量级的任务管理工具。

#### 功能

- **添加任务**：快速记录待办事项
- **状态管理**：待办 → 进行中 → 已完成，状态循环切换
- **看板视图**：切换到看板模式，三列（待办/进行中/已完成）拖拽管理
- **子任务**：在任务下添加子任务（checklist 模式）
- **标签分类**：给任务打标签，按标签筛选
- **重复任务**：设置每日/每周/每月重复，到期自动生成新任务

#### 使用场景

- 开发 TODO 清单：`添加用户登录功能` → `进行中` → `完成`
- 日常工作：每日例行任务设置重复，到点自动出现
- 项目追踪：用标签区分 `前端` / `后端` / `Bugs`

---

### 6. 定时任务（Schedules）

让 QuickDock 在指定时间**自动执行**某些操作。

#### 支持的动作类型

| 动作 | 说明 |
|------|------|
| 打开应用 | 定时启动指定应用程序 |
| 执行命令 | 定时在终端中执行命令 |
| 打开网页 | 定时在浏览器中打开网址 |
| 发送 Webhook | 定时向指定 URL 发送 HTTP 请求 |
| 待办提醒 | 到期自动创建待办任务 |

#### 重复类型

- **一次性**：指定时间执行一次
- **每天**：每天固定时间执行
- **每周**：每周某天固定时间执行
- **每月**：每月某天固定时间执行

#### 实战例子

```
每天早上 9:00   → 打开飞书/钉钉
每个周一 10:00  → 执行 git pull 拉取最新代码
每月 1 号       → 发送月报 Webhook 通知
每天 17:00      → 提醒写日报
```

### 7. 网站监控（Monitors）

监控网站或 API 的可用性，在出问题时第一时间通知你。

#### 功能

- **HTTP 探测**：定期检查 URL 是否正常返回
- **状态码验证**：可指定期望的 HTTP 状态码（如 200、301）
- **SSL 证书到期预警**：自动检测 SSL 证书到期时间，提前 X 天告警
- **内容匹配**：检查页面是否包含（或不包含）指定关键词或正则表达式
- **响应时间趋势图**：查看 24 小时 / 7 天 / 全部时段的响应时间变化
- **机器人通知**：出问题时通过 Webhook 推送到企业微信/钉钉/飞书

#### 通知内容示例

```
🔔 QuickDock 网站监控告警
监控目标: https://example.com
状态: DOWN (期望 200，实际 503)
时间: 2026-07-17 14:30:00
```

## 六、数据安全

### 快照备份

QuickDock 支持创建和恢复完整的数据快照：

1. 进入 **设置 > 快照**
2. 点击 **创建快照**，所有数据（工作空间、场景、集合、项目、片段、待办、设置等）保存为一个文件
3. 需要恢复时，点击 **从快照恢复**
4. 也可以 **删除** 不再需要的快照

> 💡 建议在重大修改前创建一个快照，作为安全网。

### WebDAV 云同步

如果你有自己的 WebDAV 网盘（如 NextCloud、Synology），可以配置云同步：

1. 进入 **设置 > WebDAV**
2. 填写服务器地址、用户名、密码
3. 点击连接测试，验证配置正确
4. 点击立即同步，数据将加密上传到你的云端
5. 另一台电脑上同样配置后，可从云端恢复

## 七、常见问题

### Q: QuickDock 是否免费？

目前免费使用。

### Q: 数据存在哪里？安全吗？

所有数据存储在本地 SQLite 数据库 `~/.quickdock/quickdock.db` 中，不上传任何云端（除非你主动配置了 WebDAV）。剪贴板中的图片存储在 `~/.quickdock/images/`。

### Q: 如何完全退出 QuickDock？

右键托盘图标 → **退出**。关闭主窗口只会隐藏到托盘，不会退出程序。

### Q: 如何备份数据？

两种方式：① 在设置中创建快照（推荐定期做）；② 配置 WebDAV 云同步（跨设备）。

### Q: 更换电脑后如何迁移数据？

1. 旧电脑：创建快照 → 导出快照文件
2. 新电脑：安装 QuickDock → 设置 > 快照 > 从快照恢复

或者直接配置 WebDAV，一键同步。

### Q: 快捷笔记的内容存在哪里？

笔记存储为一条特殊的文本片段（关键词 `__quicknote__`），与其他片段一同存放在本地数据库中。

### Q: 剪贴板历史存多少条？

默认保留 30 天（可在设置中调整）。超过保留期限或无固定标记的条目会被自动清理。
